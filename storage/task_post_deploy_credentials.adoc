= Alterar credenciais no NetApp HCI e no NetApp SolidFire
:allow-uri-read: 
:url-peak: https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/Element_Plug-in_for_vCenter_server/mNode_Status_shows_as_'Network_Down'_or_'Down'_in_the_mNode_Settings_tab_of_the_Element_Plugin_for_vCenter_(VCP)
:icons: font
:imagesdir: ../media/


[role="lead"]
Dependendo das políticas de segurança da organização que implementou o NetApp HCI ou o NetApp SolidFire, a alteração de credenciais ou senhas geralmente faz parte das práticas de segurança.  Antes de alterar as senhas, você deve estar ciente do impacto que isso terá em outros componentes de software da sua implementação.

Se você alterar as credenciais de um componente de uma implementação do NetApp HCI ou do NetApp SolidFire , a tabela a seguir fornece orientações sobre o impacto em outros componentes.

Interações entre componentes do NetApp SolidFire :image:../media/diagram_credentials_solidfire.png["Componentes NetApp SolidFire"]

[cols="10a,60a,30a"]
|===
| Tipo de credencial e ícone | Utilização pelo administrador | Consulte estas instruções. 


 a| 
Credenciais do elemento

image::../media/icon_lock_red.png[ícone de cadeado vermelho]
 a| 
*Aplica-se a*: NetApp HCI e SolidFire

Os administradores usam estas credenciais para fazer login:

* Interface de usuário do Element no cluster de armazenamento Element
* Controle de nuvem híbrida no nó de gerenciamento (mnode)


Quando o Hybrid Cloud Control gerencia vários clusters de armazenamento, ele aceita apenas as credenciais de administrador dos clusters de armazenamento, conhecidos como o _cluster autoritativo_ para o qual o mnode foi inicialmente configurado.  Para clusters de armazenamento adicionados posteriormente ao Hybrid Cloud Control, o mnode armazena com segurança as credenciais de administrador.  Caso as credenciais de clusters de armazenamento adicionados posteriormente sejam alteradas, elas também deverão ser atualizadas no mnode usando a API do mnode.
 a| 
* link:concept_system_manage_manage_cluster_administrator_users.html["Atualize as senhas de administrador do cluster de armazenamento."]
* Atualize as credenciais de administrador do cluster de armazenamento no mnode usando olink:../api/reference_element_api_modifyclusteradmin.html["API modifyclusteradmin"] .




 a| 
Credenciais de logon único do vSphere

image::../media/icon_lock_green_light.png[ícone de cadeado verde claro]
 a| 
*Aplica-se a*: Somente NetApp HCI

Os administradores usam essas credenciais para fazer login no VMware vSphere Client.  Quando o vCenter faz parte da instalação do NetApp HCI , as credenciais são configuradas no NetApp Deployment Engine da seguinte forma:

* nome de usuário@vsphere.local com a senha especificada e
* administrator@vsphere.local com a senha especificada.  Quando um vCenter existente é usado para implantar o NetApp HCI, as credenciais de Single Sign-on do vSphere são gerenciadas pelos administradores de TI da VMware.

 a| 
https://docs.netapp.com/us-en/hci/docs/task_hci_credentials_vcenter_esxi.html#%20update-the-esxi-password-by-using-the-management-node-rest-api["Atualize as credenciais do vCenter e do ESXi."^]. 



 a| 
Credenciais do controlador de gerenciamento da placa-mãe (BMC)

image::../media/icon_lock_black.png[ícone de cadeado preto]
 a| 
*Aplica-se a*: Somente NetApp HCI

Os administradores usam essas credenciais para fazer login no BMC dos nós de computação NetApp em uma implementação NetApp HCI .  O BMC oferece recursos básicos de monitoramento de hardware e console virtual.

As credenciais BMC (às vezes chamadas de _IPMI_) para cada nó de computação NetApp são armazenadas com segurança no mnode em implantações NetApp HCI .  O NetApp Hybrid Cloud Control utiliza as credenciais do BMC em uma conta de serviço para se comunicar com o BMC nos nós de computação durante as atualizações de firmware desses nós.

Quando as credenciais do BMC são alteradas, as credenciais dos respectivos nós de computação também devem ser atualizadas no mnode para manter todas as funcionalidades do Hybrid Cloud Control.
 a| 
* link:https://docs.netapp.com/us-en/hci/docs/hci_prereqs_final_prep.html["Configure o IPMI para cada nó no NetApp HCI."^].
* Para os nós H410C, H610C e H615C,link:https://docs.netapp.com/us-en/hci/docs/hci_prereqs_final_prep.html["alterar a senha padrão do IPMI"^] .
* Para nós H410S e H610S,link:task_post_deploy_credential_change_ipmi_password.html["Alterar a senha padrão do IPM"] .
* link:https://docs.netapp.com/us-en/hci/docs/task_hcc_edit_bmc_info.html["Alterar as credenciais do BMC no nó de gerenciamento"^].




 a| 
Credenciais ESXi

image::../media/icon_lock_green_dark.png[ícone de cadeado verde escuro]
 a| 
*Aplica-se a*: Somente NetApp HCI

Os administradores podem fazer login nos hosts ESXi usando SSH ou a interface de usuário do controlador de domínio local (DCUI) com uma conta root local.  Em implementações NetApp HCI , o nome de usuário é 'root' e a senha foi especificada durante a instalação inicial desse nó de computação no NetApp Deployment Engine.

As credenciais de root do ESXi para cada nó de computação NetApp são armazenadas com segurança no mnode em implantações NetApp HCI .  O NetApp Hybrid Cloud Control utiliza as credenciais em uma conta de serviço para se comunicar diretamente com os hosts ESXi durante as atualizações de firmware e verificações de integridade dos nós de computação.

Quando as credenciais de root do ESXi são alteradas por um administrador da VMware, as credenciais dos respectivos nós de computação devem ser atualizadas no mnode para manter a funcionalidade do Hybrid Cloud Control.
 a| 
link:https://docs.netapp.com/us-en/hci/docs/task_hci_credentials_vcenter_esxi.html["Atualizar credenciais para hosts vCenter e ESXi"^].



 a| 
Senha de integração de QoS

image::../media/icon_lock_purple.png[ícone de cadeado roxo]
 a| 
*Aplica-se a*: NetApp HCI e opcionalmente no SolidFire

Não é utilizado para logins interativos por administradores.

A integração de QoS entre o VMware vSphere e o Element Software é habilitada através de:

* Plug-in Element para vCenter Server e
* Serviço QoS no mnode.


Para autenticação, o serviço QoS utiliza uma senha que é usada exclusivamente neste contexto.  A senha de QoS é especificada durante a instalação inicial do plug-in Element para vCenter Server ou gerada automaticamente durante a implementação do NetApp HCI .

Não afeta outros componentes.
 a| 
link:https://docs.netapp.com/us-en/vcp/vcp_task_qossioc.html["Atualize as credenciais do QoSSIOC no plug-in NetApp Element para vCenter Server."^]. 

A senha do NetApp Element Plug-in para vCenter Server SIOC também é conhecida como senha QoSSIOC. 

Consulte o artigo da Base de Conhecimento sobre o plug-in Element para vCenter Server no link {url-peak}[].



 a| 
Credenciais do vCenter Service Appliance

image::../media/icon_lock_gray_dark.png[ícone de cadeado cinza escuro]
 a| 
*Aplica-se a*: NetApp HCI somente se configurado pelo NetApp Deployment Engine.

Os administradores podem fazer login nas máquinas virtuais do appliance vCenter Server.  Em implementações NetApp HCI , o nome de usuário é 'root' e a senha foi especificada durante a instalação inicial desse nó de computação no NetApp Deployment Engine.  Dependendo da versão do VMware vSphere implementada, determinados administradores no domínio vSphere Single Sign-on também podem fazer login no dispositivo.

Não afeta outros componentes.
 a| 
Nenhuma alteração necessária. 



 a| 
Credenciais de administrador do nó de gerenciamento NetApp

image::../media/icon_lock_gray_light.png[ícone de cadeado cinza claro]
 a| 
*Aplica-se a*: NetApp HCI e opcionalmente no SolidFire

Os administradores podem acessar as máquinas virtuais do nó de gerenciamento da NetApp para realizar configurações avançadas e solucionar problemas.  Dependendo da versão do nó de gerenciamento implantada, o login via SSH não está habilitado por padrão.

Em implementações de NetApp HCI , o nome de usuário e a senha eram especificados pelo usuário durante a instalação inicial desse nó de computação no NetApp Deployment Engine.

Não afeta outros componentes.
 a| 
Nenhuma alteração necessária.
